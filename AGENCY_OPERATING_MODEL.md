# Agency GitHub Operating Model

Updated: 2026-05-04

## Account Role

This account supports Vishnu Madhavan, 9Ruby, and IX Ruby.

Use it for:

- Public agency proof
- Client project source
- Product development
- Templates and prototypes
- Automation tooling

## Repository Buckets

Use these repo topics.

- `9ruby`: Core company work
- `ixruby`: Premium agency layer
- `client-work`: Client delivery
- `saas`: Product work
- `template`: Reusable starter
- `prototype`: Experimental build
- `vercel-backup`: Generated deployment backup
- `private`: Internal-only repo

## Public Repos

Public repos should be polished.

Required:

- Clear description
- Useful homepage
- Focused topics
- Clean README
- No secrets
- No client-private data

## Private Repos

Private repos may hold:

- Client source
- Internal tooling
- Early prototypes
- Generated backups
- Agency operations

Still required:

- Clear description
- Category topics
- No committed secrets
- Build instructions when active

## Default Files

This `.github` repo provides:

- Bug issue template
- Feature issue template
- Pull request template
- Security policy
- Contributing guide

Repo-specific files override these defaults.

## Manual Actions

These need deliberate approval:

- Creating a GitHub organization
- Transferring repositories
- Renaming repositories
- Changing visibility
- Archiving public forks
- Enforcing branch protection
- Updating profile bio scopes
